# Portico 实现计划

## 项目概述
Portico 是一个在 Linux 上运行的工具，用于建立 SSH 端口映射。用户可以通过 `portico -f xxx.yaml` 命令，根据配置文件连接到服务器并映射端口，同时显示映射端口的监听进程名称和流量。

## 项目结构
```
portico/
├── src/
│   ├── main.rs          # 主入口
│   ├── config.rs        # 配置文件解析
│   ├── ssh.rs           # SSH 连接和端口映射
│   ├── monitor.rs       # 进程和流量监控
│   └── cli.rs           # 命令行参数处理
├── Cargo.toml           # 项目配置和依赖
└── examples/
    └── config.yaml      # 配置文件示例
```

## 实现步骤

### 1. 项目初始化和依赖添加
- 添加必要的依赖包：
  - `clap`：命令行参数解析
  - `serde` + `serde_yaml`：YAML 配置文件解析
  - `ssh2`：SSH 连接和端口转发
  - `sysinfo`：进程信息获取
  - `tokio`：异步运行时
  - `tui`：终端用户界面（可选，用于显示监控信息）

### 2. 配置文件解析
- 定义配置文件结构，包括：
  - 服务器信息（主机、端口、用户名、认证方式）
  - 端口映射规则（本地端口、远程主机、远程端口）
- 实现配置文件加载和验证功能

### 3. 命令行参数处理
- 实现 `-f` 或 `--file` 参数，指定配置文件路径
- 添加其他可选参数，如 `-v` 或 `--verbose` 用于详细输出

### 4. SSH 连接和端口映射
- 实现 SSH 连接建立
- 实现本地端口转发（Local Port Forwarding）
- 处理连接错误和重连逻辑

### 5. 进程和流量监控
- 实现本地端口监听进程的识别
- 实现流量监控功能
- 设计监控数据的存储和更新机制

### 6. 监控界面
- 实现终端界面，显示：
  - 连接状态
  - 映射端口列表
  - 每个端口的监听进程名称
  - 每个端口的流量统计
- 支持实时更新和交互操作

### 7. 错误处理和日志
- 实现全面的错误处理
- 添加日志记录功能
- 确保程序的稳定性和可靠性

### 8. 测试和优化
- 编写单元测试和集成测试
- 优化性能和资源使用
- 确保在不同 Linux 环境下的兼容性

## 技术栈
- **语言**：Rust
- **异步运行时**：Tokio
- **SSH 库**：ssh2
- **配置解析**：serde + serde_yaml
- **命令行解析**：clap
- **进程信息**：sysinfo
- **终端界面**：tui (可选)

## 预期功能
1. 支持通过 YAML 配置文件定义多个服务器和端口映射
2. 自动建立 SSH 连接并设置端口转发
3. 实时监控映射端口的状态和流量
4. 提供友好的终端界面显示监控信息
5. 支持后台运行和日志记录

## 风险评估
1. **SSH 连接稳定性**：需要处理网络波动和连接中断的情况
2. **权限管理**：确保程序有足够的权限进行端口绑定和进程监控
3. **性能开销**：监控功能可能会增加系统资源使用，需要优化
4. **跨平台兼容性**：虽然目标是 Linux，但需要确保在不同发行版上的兼容性
